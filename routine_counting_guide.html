2. Step-by-Step: Routine Count Process

Step 1 — Preparation (Before Counting)

  • Stop all movement — No receiving, transfers, wastage, or production during the count
  • Organize storage — Everything should be in its assigned storage area
  • Assign counters — Ideally, have someone count who is NOT the person responsible for the stock (separation of duties)
  • Print/open count sheets — Use the system's Stock Count session

Step 2 — Start a Count Session (In the System)

  1. Go to Inventory → Stock Management → Stock Count
  2. Click Start New Count
  3. Select the Business Unit
  4. The system loads all active items with their theoretical stock (what the system thinks you have)

Step 3 — Physical Counting Rules

📋 Golden Rules of Counting
RuleWhy
Count everything in recipe units (the base unit in the system)Ensures consistency — the system stores stock in recipe units
Convert as you go — e.g., 2 cases of beer (12 bottles each) = 24 bottlesIf recipe unit is "bottle", enter 24, not 2
Count full units first, then partialsFor liquids: full bottles + estimate partial bottles (e.g., 0.5 for half full)
Never round — be preciseA variance of 0.5 bottles matters over time
Count twice if unsureBetter slow and accurate than fast and wrong
Count ALL locations for the same itemCheck bar, kitchen, storage room — combine totals

3. How to Count Each Item Type

🧴 Raw Materials (Ingredients)

Example: "Tanqueray Gin" Recipe Unit: ml | Buy Unit: bottle (750ml) | Conversion: 750 Physical count: 3 full bottles + 1 bottle ~60% full Calculation: (3 × 750) + (0.6 × 750) = 2,250 + 450 = 2,700 ml Enter: 2700

🍳 Production Items (Pre-batched mixes, sauces)

Example: "House BBQ Sauce" Recipe Unit: ml | Yield per batch: 2000ml Physical count: 1 full container (2000ml) + 1 container ~30% full Calculation: 2000 + (0.3 × 2000) = 2,600 ml Enter: 2600

🏷️ Finished Goods (Items you sell)

Example: "Bottled Hot Sauce (250ml)" Recipe Unit: EA (each) Physical count: 14 bottles on shelf + 1 case (12) in storage Enter: 26

📦 Dry Goods / Supplies

Example: "Paper Cups" Recipe Unit: EA Physical count: ~45 cups in open sleeve + 2 full packs (50 each) Enter: 145

4. Entering Counts & Reviewing Variances

After entering all counts, focus on variance analysis:

VarianceMeaningAction
0 or very small (±1-2%)Normal✅ Accept
Negative (missing stock)Possible theft, unrecorded wastage, or missed receiving⚠️ Investigate
Positive (extra stock)Possible receiving not entered, or miscount last time⚠️ Investigate
Large % on high-value itemsSignificant financial impact🚨 Immediate investigation
⚠️ Red Flags to Watch
  • Same item consistently short every count → possible theft pattern
  • Spirits short by exactly 1 bottle increments → likely pilferage
  • Raw materials short but no corresponding production recorded → unrecorded wastage or recipes not being followed

Finalize the Count

  1. Review all variances
  2. Add notes for large discrepancies (e.g., "Found 2 bottles broken in storage, not logged as wastage")
  3. Submit/Finalize the count session
  4. The system updates currentStock to match your physical count
  5. theoreticalStock resets to equal currentStock (clean baseline for next period)

6. Anti-Theft Best Practices

PracticeImplementation
Separation of dutiesPerson counting ≠ person managing that stock
Blind countsCounter should NOT see the theoretical stock before counting
Rotate countersDon't let the same person count the same area every time
Log ALL wastageEvery broken bottle, spoiled ingredient, or spill → Wastage module
Record ALL receivingEvery delivery → Goods Receiving module, link to PRF
Review variance reportsManagement reviews weekly, signs off
Camera coverageStorage areas should have camera coverage
Lock high-value stockSpirits + expensive ingredients in locked storage, key held by manager

7. Common Mistakes to Avoid

❌ Mistake✅ Correct Approach
Counting in buy units (cases) and entering as-isConvert to recipe units before entering
Counting during active serviceCount when operations are stopped
Skipping partial bottles/containersEstimate partials — every bit counts
Forgetting items in secondary locationsCheck ALL storage areas for every item
Not recording wastage between countsLog wastage immediately when it happens
Counting production items but not deducting ingredientsProduction module handles ingredient deduction — just count what's physically there
Adjusting theoretical stock manually to hide variancesNever — the variance IS the data you need

📌 The Counting Rhythm — Summary

📅 DAILY
  • Record all wastage immediately
  • Record all receiving via Goods Receiving
  • Spot count high-value items (optional)
📅 WEEKLY (Sunday)
  • Full physical count of all items
  • Enter counts in Stock Count session
  • Review variances, investigate anomalies
  • Finalize → system resets baseline
📅 MONTHLY
  • Review variance trend reports
  • Identify repeat problem items
  • Adjust par levels if needed
  • Management sign-off

Count what's real, let the system show you the gaps.
The variance between theoretical and actual is where all your operational insights live — don't hide it, investigate it.
